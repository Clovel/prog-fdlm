# Visitor-friendly `/[year]` 404 Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the bare `(public)/[year]` not-found page with a polished, music-themed 404 (custom inline SVG + friendly French copy + a single "back to home" button), and route malformed years to it.

**Architecture:** A small server-component SVG illustration (`Music404.tsx`) is rendered by a rewritten server-component `not-found.tsx`, both inside the existing public layout (Header + footer). The client `page.tsx` gains an early `notFound()` guard for non-4-digit years and treats API `400` like `404`, so every not-found case lands on the friendly page.

**Tech Stack:** Next.js App Router (server components), React 19, Tailwind v4 (theme-token utilities `fill-*`), shadcn `Button`, `next/link`.

**Testing note:** This repo has **no test framework** (per `CLAUDE.md`). Verification per task is `pnpm tsc:ci`, `pnpm lint`, and a visual dev-server check — there are no unit tests to write. Follow the repo's ESLint conventions exactly: 2-space indent, single quotes in JS / double in JSX, semicolons, always-multiline trailing commas, **no space after `if`** (`if(x)`), `strict-boolean-expressions` (no truthy checks on strings — compare explicitly), explicit return types on non-trivial arrows, `import type` for type-only imports, and the component banner comment layout used across the repo.

### Task 3: Route malformed years to the 404

**Files:**
- Modify: `src/app/(public)/[year]/page.tsx`

The current fetch helpers call `notFound()` only on `status === 404`. The component reads `year` from params but never validates its shape, so `/banane` returns API `400` → throws → generic error. Two edits.

- [ ] **Step 1: Widen the 404 check in both fetch helpers**

In `fetchEdition` (around line 47) change:

```tsx
  if(response.status === 404) {
    notFound();
  }
```

to:

```tsx
  if(response.status === 404 || response.status === 400) {
    notFound();
  }
```

Apply the identical change in `fetchEvents` (around line 60).

- [ ] **Step 2: Add an early synchronous guard in the component body**

In `EditionPage`, immediately after the two lines:

```tsx
  const params = useParams<{ year: string }>();
  const year: string = params.year;
```

insert:

```tsx

  if(!/^\d{4}$/.test(year)) {
    notFound();
  }
```

`notFound()` is already imported from `next/navigation` at the top of the file — no new import needed. Calling it during render of a client component throws the not-found signal synchronously, so malformed years never reach the doomed fetch or the error flash.
